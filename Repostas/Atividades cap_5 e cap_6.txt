Atividade 1:

Código em Python:
x = 10  # variável global

def f():
    print("Valor de x em f():", x)

def g():
    x = 20  # variável local de g()
    f()    # chama f()

g()

Código em JavaScript:
let x = 10; // variável global

function f() {
    console.log("Valor de x em f():", x);
}

function g() {
    let x = 20; // variável local de g()
    f();        // chama f()
}

g();

Reflexão:
O valor impresso depende do local de definição da função, não do local da chamada.
Ambas (Python e JavaScript) seguem escopo estático. Linguagens com escopo dinâmico usariam o valor de x do chamador.

Atividade 2:

Código em C:
#include <stdio.h>

void contador() {
    int a = 0;       // variável automática (reinicia sempre)
    static int b = 0; // variável estática (mantém valor)
    
    a++;
    b++;
    printf("a = %d, b = %d\n", a, b);
}

int main() {
    contador();
    contador();
    contador();
    return 0;
}

Reflexão:
a é recriada a cada chamada, por isso sempre começa em 0.
b mantém seu valor entre chamadas, pois está em memória estática.Isso ilustra a diferença entre tempo de vida automático e tempo de vida estático.

Atividade 3:

Código em Java:
public class TesteTipos {
    public static void main(String[] args) {
        int num = 10;
        // num = "dez";  // ERRO: tipos incompatíveis (não compila)
        
        System.out.println(num + 5); // OK -> imprime 15
    }
}


Código em Python:
num = 10
print(num + 5)  # 15

num = "dez"
# print(num + 5)  # ERRO em tempo de execução: não é possível somar str e int


Reflexão:
Java não permite atribuir string a int. Tipagem é estática e verificada em tempo de compilação.
Python permite trocar o tipo da variável em tempo de execução (tipagem dinâmica), mas operações inválidas geram erro em tempo de execução.

Vantagens da tipagem estática: segurança, performance, prevenção de bugs antes da execução. Vantagens da tipagem dinâmica: flexibilidade, código mais curto, prototipagem rápida.

Atividade 4:

Código em C:
#include <stdio.h>

struct Livro {
    char titulo[50];
    char autor[50];
    int anoPublicacao;
};

int main() {
    int numeros[5] = {1, 2, 3, 4, 5};  // array homogêneo
    printf("Array: ");
    for (int i = 0; i < 5; i++) {
        printf("%d ", numeros[i]);
    }

    printf("\n");

    struct Livro livro1 = {"O Hobbit", "J.R.R. Tolkien", 1937};
    printf("Livro: %s, Autor: %s, Ano: %d\n", livro1.titulo, livro1.autor, livro1.anoPublicacao);

    return 0;
}


Código em Java:
import java.util.ArrayList;

class Livro {
    String titulo;
    String autor;
    int anoPublicacao;

    Livro(String t, String a, int ano) {
        titulo = t;
        autor = a;
        anoPublicacao = ano;
    }
}

public class Main {
    public static void main(String[] args) {
        ArrayList<Livro> lista = new ArrayList<>();
        lista.add(new Livro("O Hobbit", "Tolkien", 1937));
        lista.add(new Livro("Dom Casmurro", "Machado de Assis", 1899));
        lista.add(new Livro("1984", "George Orwell", 1949));

        System.out.println("Títulos adicionados:");
        for (Livro l : lista) {
            System.out.println(l.titulo);
        }
    }
}


Reflexão:
Um array guarda vários valores do mesmo tipo em ordem, enquanto um registro/objeto agrupa diferentes tipos de dados em uma única entidade. Usa-se array para listas homogêneas e struct/classe para representar objetos com vários atributos.
